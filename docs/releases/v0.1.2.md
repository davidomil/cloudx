# Cloudx v0.1.2

Cloudx v0.1.2 turns the automation builder into an executable workflow
surface. It adds bounded Python, Bash, sleep, and Codex execution primitives,
test fixtures for automation groups, stronger hook execution controls, and
operator-visible persistence status when workspace or automation state cannot
be saved.

## What's New

- Added Python, Bash, sleep, and Codex execution primitives to automation
  graphs.
- Added automation group test cases with saved payloads and expected run
  assertions.
- Added dynamic automation catalog options, broader catalog coverage, and
  documentation for code execution behavior.
- Added shared AI model option metadata for server and web configuration.
- Added a reusable local test-environment setup script.

## Security And Reliability

- Hardened Python automation hook calls against spoofed stdout control
  messages.
- Isolated Python and Bash automation subprocesses from the full server
  environment.
- Strengthened automation subprocess timeout and cancellation behavior by
  terminating process groups with a forced-kill grace period.
- Applied automation hook safety checks to Python `cloudx.call_hook` requests.
- Passed HTTP request abort signals into normal and streaming hook calls.

## Improvements And Fixes

- Forwarded automation test-case inputs through plugin and HTTP hook paths.
- Refreshed clean cached automation panels on remount while preserving dirty
  drafts.
- Displayed degraded workspace or automation persistence status in the app
  topbar.
- Improved Jira issue creation/update handling and plugin hook coverage.
- Improved documentation ingest, enrichment, and plugin action behavior.
- Improved terminal readiness handling and cancellation coverage.
- Improved workspace state persistence and notification behavior.

## Validation

- Focused automation, plugin, server, App, and Automation Panel tests pass.
- Full Vitest suite passes.
- TypeScript project build passes.
- Production web/server build passes.
- Git whitespace checks pass.

## Install

```bash
git clone https://github.com/davidomil/cloudx
cd cloudx
./install.sh
```

Cloudx remains private by default and binds to `127.0.0.1`. Use
`./install.sh --lan` only for a trusted LAN or tailnet. Public internet
exposure is unsupported.
