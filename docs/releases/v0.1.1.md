# Cloudx v0.1.1

Cloudx v0.1.1 focuses on operating the local workbench across more real
installations: runtime diagnostics, plugin installation visibility, Jira and
automation workflows, documentation archive tooling, and hardening across
server, terminal, workspace, and browser surfaces.

## What's New

- Added the Jira integration plugin and Jira automation triggers.
- Added documentation archive ingest, search, source viewing, enrichment,
  stale invalidation, export, import, spreadsheet ingest, media ingest, and
  schematic-aware artifacts.
- Added plugin-contributed rule and skill injection for Codex terminal
  contexts.
- Added Codex terminal image paste support.
- Added the automation workflow builder.
- Added GitHub plugin metadata installation support.

## Diagnostics

- Added `CLOUDX_LOG_LEVEL` for runtime logging control. Set it to `debug` or
  `trace` in the Cloudx environment file when collecting diagnostics.
- Added debug logging for installed plugin catalog loading, GitHub plugin
  installation phases, plugin registration, and plugin rule/skill contribution
  sync.
- Expanded installer verbose diagnostics while keeping sensitive environment
  values out of the verbose output.

## Improvements And Fixes

- Hardened server routes, websocket handling, state storage, automation
  execution, terminal runtime, ASR/app-server lifecycle, local web proxying,
  plugin webview bridging, and web client workspace behavior.
- Improved documentation archive locality, payload bounds, stats bounds, source
  read bounds, artifact payload size, path ingest resolution, and lazy document
  loading.
- Improved file browser previews, transfer controls, toolbar placement, diff
  limits, and search behavior.
- Improved mobile terminal viewport handling, reconnect layout, mobile keyboard
  viewport behavior, and terminal focus handling.
- Improved installer service setup, optional Whisper.cpp setup guidance, and
  bootstrap package coverage.

## Validation

- Full Vitest suite passes.
- TypeScript project build passes.
- Production web/server build passes.
- Startup smoke test passed on `127.0.0.1:3201`.
- Interactive browser smoke test passed for loading the app, creating a Files
  tab, browsing the repository tree, running file search, and loading a mobile
  viewport.

## Install

```bash
git clone https://github.com/davidomil/cloudx
cd cloudx
./install.sh
```

Cloudx remains private by default and binds to `127.0.0.1`. Use
`./install.sh --lan` only for a trusted LAN or tailnet. Public internet
exposure is unsupported.
